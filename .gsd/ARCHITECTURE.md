## Overview

ChanakyaOS is a Next.js 16 (App Router) based monorepo application. It follows a hybrid rendering strategy (SSR for marketing, CSR for dashboard) and integrates deeply with Supabase for data and Google Gemini for AI.

```
┌─────────────────────────────────────────┐
│        Next.js App Router (UI)          │
├─────────────────────────────────────────┤
│    Next.js API Routes / Server Actions  │
├─────────────────────────────────────────┤
│    Supabase (Auth, DB, Vector, Storage) │
└─────────────────────────────────────────┘
```

## Components

### Frontend (src/app)
- **Marketing**: Pre-login pages (Home, About, Careers, Contact).
- **Dashboard**: Post-login user experience (Overview, Pathway, Resume).
- **Onboarding**: Multi-step wizard for profile building.

### UI Components (src/components)
- **Marketing Components**: Specialized UI for landing pages.
- **LMS Components**: Reusable widgets for pathway tracking and skill analysis.
- **Providers**: SyncProvider (hardened), ThemeProvider, AuthProvider.

### Logic Layer
- **Sync Engine**: Centralized data persistence layer in `SyncProvider`.
- **AI Pipelines**: Route handlers in `src/app/api/ai` for Gemini interactions.

## Data Flow

1. **Onboarding**: User inputs data → AI parses resume → Supabase stores Professional Identity Map.
2. **Diagnosis**: Profile data → Gemini analyzing against benchmarks → Diagnosis stored in Supabase.
3. **Pathway**: Goals/Skills → Gemini generating paths → Paths stored and tracked via `SyncProvider`.

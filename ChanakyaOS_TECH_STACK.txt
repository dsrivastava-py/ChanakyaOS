================================================================================
  ARCHITECTURE OVERVIEW
================================================================================

  · Public landing + marketing pages (SEO-critical, SSR)

================================================================================
  SECTION 1 — FRONTEND FRAMEWORK
================================================================================

  PRIMARY: Next.js 15 (App Router)
  Version: 15.x (latest stable as of 2026)
  License: MIT (Free)
  Cost: $0

  WHY NEXT.JS 15 FOR CHANAKYA OS:
  ─────────────────────────────────────────────────────────────────────────────
  1. SEO CRITICAL — Landing pages, blog, and public career content need SSR
     for Google indexing. Indian users heavily discover via search. Next.js
     App Router handles SSR + ISR natively.

  2. HYBRID RENDERING — Dashboard pages (CSR) + marketing pages (SSR) in
     one codebase. No separate frontend/backend split needed at Phase 1.

  3. API ROUTES — Built-in serverless API routes eliminate need for a
     separate backend server in Phase 1–2, keeping infra cost at $0.

  4. APP ROUTER — React Server Components reduce JS bundle size. Streaming
     responses match AI-powered features (Suspense + streaming AI output).

  5. TURBOPACK — Development build speed 5–10x faster than Webpack.
     Critical for a complex dashboard with many components.

  6. IMAGE OPTIMIZATION — next/image handles WebP conversion and lazy
     loading automatically. Important for Indian users on slower connections.

  7. PWA READY — next-pwa plugin adds service worker + offline support for
     Phase 1 mobile-first requirement at zero cost.

  8. REACT ECOSYSTEM — Full access to React 19 features: Actions, use(),
     optimistic updates. Perfect for the AI assistant chat interface.

  9. VERCEL DEPLOYMENT — Free hobby tier on Vercel. Zero-config deployment
     with edge functions and CDN included.

  KEY NEXT.JS 15 FEATURES TO USE:
  · Server Components for data-fetch-heavy dashboard sections
  · Client Components for interactive widgets (score ring, pathway cards)
  · Parallel Routes for the split dashboard layout (sidebar + main)
  · Intercepting Routes for modal flows (career pathway detail)
  · Server Actions for form submissions (onboarding wizard, profile updates)
  · Streaming with Suspense for AI chat responses (word-by-word output)
  · Middleware for auth guards on protected routes

  REACT VERSION: React 19 (bundled with Next.js 15)

  FREE ALTERNATIVE (if Next.js complexity is overkill for MVP):
  → Remix v3 — similar SSR capabilities, simpler mental model
  → SvelteKit 2 — smaller bundle, faster runtime, steeper learning curve
  → Recommendation: Stick with Next.js 15. Ecosystem + Vercel free tier win.

  1. DESIGN DOC ALIGNMENT — The ChanakyaOS design doc already defines exact
     CSS custom properties (:root tokens). Tailwind v4 natively supports
     CSS variables as design tokens — direct 1:1 mapping to the design system.

  2. DARK-FIRST DESIGN — Entire platform is dark (#0B0F19 background).
     Tailwind dark mode config eliminates toggle complexity.

  3. V4 ENGINE — Lightning CSS-based engine (Rust). No PostCSS overhead.
     Instant HMR in development. 100x faster than v3 for large projects.

  4. ZERO RUNTIME — Pure utility classes. No CSS-in-JS runtime overhead.
     Critical for performance on Indian mobile networks.

  5. BENTO GRID SYSTEM — Tailwind's grid utilities (col-span, row-span)
     map directly to the bento grid layouts defined in the design doc.

  6. GLASSMORPHISM SUPPORT — backdrop-blur, bg-opacity utilities handle
     all glass card effects specified in the design doc natively.

  4B. PGVECTOR (Vector Database — REPLACES Pinecone)
  ────────────────────────────────────────────────────────────────────────────

  WHAT IT DOES:
  · Stores semantic embeddings of user profiles and career data
  · Powers semantic search across pathways, skills, certifications
  · Enables "find similar career paths to yours" feature
  · RAG (Retrieval Augmented Generation) for context-aware AI responses

  HOW TO USE:
  -- Enable pgvector in Supabase
  CREATE EXTENSION IF NOT EXISTS vector;

  -- User profile embedding for similarity search
  ALTER TABLE user_profiles ADD COLUMN embedding vector(768);

  -- Career pathway embeddings
  ALTER TABLE career_pathways ADD COLUMN embedding vector(768);

  -- Similarity search query
  SELECT * FROM career_pathways
  ORDER BY embedding <=> query_embedding
  LIMIT 5;

  EMBEDDING MODEL: all-MiniLM-L6-v2 via HuggingFace Inference API (FREE)
  Dimensions: 384 (smaller, faster) or 768 (higher quality)

  · Resume optimization feedback
  · LinkedIn profile analysis
  · Skill gap analysis
  · Market trend summarization

  SDK: @google/generative-ai (npm) — official Google package, free

  STREAMING IMPLEMENTATION:
  const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY);
  const model = genAI.getGenerativeModel({ model: "gemini-2.0-flash" });
  const result = await model.generateContentStream(prompt);
  for await (const chunk of result.stream) {
    process.stdout.write(chunk.text());
  }

  5E. RESUME PARSING (Free Alternative to AWS Textract)
  ────────────────────────────────────────────────────────────────────────────

  STRATEGY: Two-stage parsing pipeline — completely free

  Stage 1 — PDF Text Extraction:
  TOOL: pdfplumber (Python, MIT License, $0)
  · Extracts raw text + layout from PDF resumes
  · Handles multi-column Indian resume formats
  · Extracts tables (education, experience) with structure intact

  Stage 2 — NLP Entity Extraction:
  TOOL: Gemini 2.0 Flash (multimodal vision API, FREE)
  · Send extracted text to Gemini with structured extraction prompt
  · Returns JSON: {name, email, phone, skills[], education[], experience[],
    projects[], certifications[], github_url, linkedin_url}
  · Handles Indian name formats, Indian university names, Indian company names
  · Far more accurate than spaCy NER for resume-specific entities

  BACKUP PARSING: spaCy (en_core_web_sm — free, open source)
  · Rule-based fallback for when AI quota exhausted
  · Custom NER pipeline for Indian resume entities

  -- Users can only read/write their own data

  SECTION 8 — PDF GENERATION (Resume Builder)
================================================================================

  PRIMARY: @react-pdf/renderer
  Version: 4.x (latest)
  License: MIT (Free)
  Cost: $0

  WHY REACT-PDF OVER PUPPETEER:
  · Client-side PDF generation — no server needed (Puppeteer needs a server)
  · React components describe the PDF layout — reuses design system
  · Zero server cost — runs in browser or in Next.js API routes serverlessly
  · Supports custom fonts (Clash Display, Satoshi, Inter — exact match to UI)
  · Generated PDFs are pixel-perfect and ATS-parseable

  IMPLEMENTATION:
  // Define resume as React PDF components
  import { Document, Page, Text, View, StyleSheet } from '@react-pdf/renderer';

  const Resume = ({ data }) => (
    <Document>
      <Page style={styles.page}>
        <View style={styles.header}>
          <Text style={styles.name}>{data.name}</Text>
        </View>
        {/* ... rest of resume sections */}
      </Page>
    </Document>
  );

  // Generate PDF blob in browser
  const blob = await pdf(<Resume data={userData} />).toBlob();

  ALTERNATIVE (for complex layouts):
  · html2canvas + jsPDF — simpler but lower quality
  · Puppeteer on Render.com free tier — higher quality but server required
  → Recommendation: @react-pdf/renderer for ATS resumes. Use Puppeteer
    only if pixel-perfect visual resume templates are needed in Phase 2+.

  SECTION 16 — SECURITY BEST PRACTICES
================================================================================

  1. ENVIRONMENT VARIABLES — Never expose API keys to client-side code
     · GEMINI_API_KEY — server-only, proxied through /api/ai/* routes
     · SUPABASE_SERVICE_ROLE_KEY — server-only, never in browser
     · Use NEXT_PUBLIC_ prefix ONLY for truly public values

  2. ROW LEVEL SECURITY — All Supabase tables have RLS policies enabled
     · Users can only read/write their own data
     · Service role key bypasses RLS — only use in server-side code

  3. RATE LIMITING — Protect AI APIs from abuse
     · Upstash Redis rate limiter on all /api/ai/* routes
     · Free users: 15 AI requests/hour
     · Pro users: 100 AI requests/hour
     · Premium: 500 AI requests/hour

  4. INPUT VALIDATION — Zod schemas on all API endpoints
     · Validate + sanitize all user inputs before passing to LLM
     · Prompt injection protection: strip HTML + special chars from inputs

  5. CORS — Restrict API routes to own domain
     · next.config.js: allowedOrigins: ['https://chanakyaos.com']

  6. FILE UPLOAD SECURITY
     · Max file size: 5MB for resumes
     · Allowed types: application/pdf, application/msword only
     · Scan with VirusTotal API (free tier, 500 scans/day) before processing
